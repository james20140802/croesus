# Research Agent — Local-LLM Notes on Rebalance Proposals

Sprint 010 attaches a qualitative research note (business / catalysts /
risks) to every rebalance action flagged `requires_research`, generated by a
**local** model. Zero API cost, and portfolio data never leaves the machine.

## Setup

Any OpenAI-compatible launcher works; only the base URL differs:

| Launcher | `CROESUS_LLM_BASE_URL` |
|---|---|
| Ollama (default) | `http://localhost:11434/v1` |
| LM Studio | `http://localhost:1234/v1` |
| llama.cpp server | `http://localhost:8080/v1` |
| vLLM / MLX servers | their `/v1` endpoint |

```bash
export CROESUS_LLM_MODEL="qwen3:32b"     # default; use 14b/8b on smaller hardware
export CROESUS_LLM_TIMEOUT=180           # seconds per note (default)
ollama pull qwen3:32b && ollama serve    # once
```

Notes are generated inside `rebalance_check` — no separate job. They are
persisted to `research_notes` (with the generating `model` recorded) and
rendered in the portfolio action report's `## Research Notes` section.

## Failure policy (never blocks the pipeline)

- **Server down / wrong URL / model not pulled** → one warning with a
  diagnostic (`ollama serve`, `ollama pull <model>`), the rebalance run
  completes without notes.
- **One asset's response unusable** → that note is stored as `failed` with
  the error; remaining assets continue.

## Honest limitations (designed in, not papered over)

- The model has **no web access** and a fixed training cutoff. Prompts
  therefore instruct it to interpret the pipeline's own quantitative evidence
  (factor sub-scores, raw multiples, DCF assumptions, macro regime) and to
  frame anything time-sensitive as *items for the human to verify*. Every
  note carries `knowledge_cutoff_caveat` and the report section repeats the
  warning.
- **Notes never propose trades.** The system prompt forbids it, the schema
  has no quantity/price/side columns, and the agent runs after actions are
  already persisted — there is no code path from a note to an action.
- Under `Cautious`/`Defensive` macro positioning, a deterministic
  `[Macro warning]` line is prepended to the risks — code, not the LLM,
  guarantees that warning.

## Manual verification

```python
from croesus.db.connection import get_connection

with get_connection() as conn:
    print(conn.execute("""
        SELECT asset_id, model, status, substr(business_summary, 1, 80)
        FROM research_notes ORDER BY created_at DESC LIMIT 5
    """).df())
```
