# 수익 창출 리서치 엔진 — 다중 방법론 — Design Spec
_2026-06-15_

## Summary

시스템의 목적을 **위험 관리 → 수익 창출**로 확장한다.

기존 포트폴리오 관리는 위험 관리였다. composite_v1로 "이게 실제로 위험을 줄여준다"는 건
확인됐지만(Sharpe 개선), **시장을 이기지는 못한다** — composite_v1 +13.1% CAGR vs SPY
14.3%, 개선된 건 위험조정 지표뿐. 사용자가 원하는 건 **수익**: 단타가 아니라 긴 호흡으로,
**지금은 제대로 평가받지 못하지만 미래에 성공할 "모래 속 진주"** 를 찾는 것(과거의 애플·
구글처럼). 멀티년 예측이 어려우면 **더 짧은 단위의 리프라이싱**(정책 지원, 경쟁구도 변화
등 forward 촉매)도 노린다.

이 설계는 그 수익 소싱을 **단일 정답이 아니라 다중 방법론 메뉴**로 만든다. 여러 리서치
방법론(아래 §방법론 A/B …)이 공통 기반 위에서 각자 기회를 산출하고, **사용자가 어떤 방법을
쓸지 선택**한다 — 이미 forward-test가 여러 weight scheme을 나란히 두는 패턴과 동형이다.

**경계선:** 모든 방법론에서 LLM은 증거에 묶인 "판단·추천"까지만 한다. 숫자와 실행 책임은
각각 결정론적 코드와 사람의 몫이다.

---

## Background

### 핵심 재정의 — 위험관리(완료) + 수익창출(신규)의 공존

- **기존 팩터 스크린 + 포트폴리오 위험 레이어 = 하방을 막는 게이트.** 우리가 "위험이
  관리된다"고 확인한 그것. **그대로 둔다.**
- **신규 기회 엔진 = 상방을 길어오는 소싱 레인.** 사용자가 원하는 수익 창출.
- 즉 **기회 엔진이 상방을 찾고, 포트폴리오 레이어가 하방을 지킨다.** 대체가 아니라 두 반쪽.

### "모래 속 진주"가 강제하는 것

진주의 정의상 그건 **현재 팩터 스크린이 높게 안 잡는 종목**이다 — 모멘텀·퀄리티는 후행
데이터라, 아직 안 뜬 미래 승자나 촉매 직전 종목은 점수가 낮다. 따라서 기회 엔진은 **팩터
숏리스트에 주석만 달아선 진주를 구조적으로 못 찾고**, 넓은 유니버스를 **싼 결정론적
전처리**로 훑어 후보를 새로 길어와야 한다.

### 두 종류의 수익 thesis

사용자 예시는 두 호라이즌으로 갈린다:
1. **구조적 승자** (해자·기술·산업 유망성) — 멀티년, 애플/구글형.
2. **촉매 리프라이싱** (정책·규제·경쟁구도 변화) — 수개월~2년, "더 짧은 단위".

### 철학 정합성과 긴장

철학 §3은 LLM 용도로 *"News, Earnings transcripts, SEC filings, Competitive positioning,
Product and industry narratives, Regulatory risk"* 를 명시 → 이 설계가 §3을 구현한다.
그러나 §133은 *"Pure LLM-driven stock picking", "Meme-stock recommendations"* 를 금지한다.
**해소:** 증거 강제 + 사람 게이트(§가드레일)로 "증거 기반 추천"이지 "vibes 종목추천"이
아니게 한다. LLM은 산업 전망에서 자신만만하게 틀리고 낙관 편향이 심함을 전제로 설계한다.

### 다중 방법론 원칙

수익 소싱에 단일 정답은 없다. 시스템은 **여러 방법론을 나란히** 제공하고 사용자가 고른다.
방법론들은 공통 기반(수집·LLM경계·위험게이트)을 공유하고, 출력은 비교 가능한 형식
(기회 후보 + 방향 + 호라이즌 + 근거 + confidence)으로 정규화한다.

## Architecture

### 공통 기반 (모든 방법론 공유)

```
[수집]  SEC EDGAR(1차 공시: 10-K/10-Q/8-K, XBRL) + 뉴스 API(보조)
[경계]  LLM은 증거에 묶인 판단/추출만 → 숫자는 결정론적 코드 → 사람 검수
[게이트] 기존 포트폴리오/위험 레이어가 집중도·유동성·프로파일 제약으로 최종 필터
```

### 후보 소싱 (방법론 공유) — 넓은 다스캔 + 싼 전처리

```
전체 유니버스
   ↓  싼 결정론적 전처리 (LLM 없음) ← 진주를 거르는 깔때기, 질이 전부를 결정
[이벤트·이상 트리거]  신규 8-K/공시, 가이던스 변경, 뉴스 급증,
                      이상 거래량·수익률, 밸류 디스로케이션
   ↓  "여기 뭔가 forward한 일이 벌어졌다" 후보군 (팩터 스크린이 안 잡는 종목 포함)
   → 각 방법론이 이 후보군에 LLM thesis 적용
```

**전처리 설계 = 이벤트 드리븐.** 조용한 데이터에서 진주를 예측하려 들면 실패한다. "방금
뭔가 벌어진 곳"(공시·뉴스·이상거래)을 깔때기로 쓰면 사용자 예시(정책·경쟁사 동향)와 맞고
LLM 비용도 소수에만 든다. 밸류 디스로케이션(저렴 멀티플)은 보조 트리거.

### 방법론 A — 해자반영 내재가치 (DCF 환원)

기업 가치를 forward 정성 판단으로 보정한 내재가치와 현재가를 비교한다. 펀더멘털이 탄탄한
**구조적 승자형**에 적합.

- **현재가치 DCF (닻):** 기존 `two_stage_dcf`, 손잡이 기계적 기본값.
- **구조적 thesis (LLM, 이산 등급 + 증거):**

  | 차원 | 등급 | 근거(공시에서) |
  |---|---|---|
  | 경쟁우위/해자 | wide/narrow/none | 전환비용·네트워크효과·규모·IP·브랜드 |
  | 기술력 | leading/parity/lagging | R&D 집약도·제품 로드맵·점유율 |
  | 분야 유망성 | secular_growth/stable/declining | TAM 성장·산업 데이터 |
  | 파괴 리스크 | low/medium/high | 신기술·신규진입자·규제 |

- **등급 → 장기 DCF 손잡이 (우리가 소유한 고정·bounded 테이블):**

  ```python
  CAP_YEARS        = {"wide": 10, "narrow": 7, "none": 5}              # 현재 고정 5년
  TERMINAL_GROWTH  = {"secular_growth": 0.030, "stable": 0.025, "declining": 0.015}
  RISK_PREMIUM     = {"low": 0.00, "medium": 0.01, "high": 0.02}       # WACC 가산
  ```
  LLM은 등급만, 등급→숫자는 감사가능한 테이블. 기존 캡(`FCF_GROWTH_CAP` 등)으로 바운드.
- **근단기 정량 주장:** 가이던스 숫자를 출처와 함께 추출 → 단기 성장률에 대입·클램프(보조).
- **출력:** `현재가 | 현재가치 DCF | 해자반영 DCF (bear/base/bull)` 밴드 + thesis 카드.

### 방법론 B — 이벤트 드리븐 기회 thesis

펀더멘털 모델로 환원하기 어려운 **촉매 리프라이싱형**에 적합. 후보의 실제 공시·뉴스를
LLM이 읽고 "시장이 아직 반영 못 한 forward 사유"를 thesis로 구조화.

- **출력(정규화):** thesis 유형(구조적/촉매) + 방향(상/하) + 대략 호라이즌 + 근거·출처 +
  confidence + **무엇이 틀리게 하나(bear)**.
- DCF 환원이 가능하면 방법론 A의 밴드를 보조로 첨부, 불가하면 정성 thesis만.

### 방법론은 확장 가능

A/B는 시작점이다. 이후 다른 방법론(예: 어닝콜 transcript 톤, 섹터 로테이션 신호 등)을 같은
기반·출력 형식으로 추가할 수 있다.

### 사용자 선택 + 사람 게이트

각 방법론이 기회 리스트를 산출 → **사용자가 어떤 방법론을 쓸지/어떻게 가중할지 선택** →
선택된 후보가 기존 포트폴리오/위험 게이트(집중도·유동성·프로파일)를 통과해야 실제 후보 →
**사람이 최종 결정·실행**. 자동 매수/매도 없음(철학 §7).

### 가드레일 (§133 vibes-picking 방지, 전 방법론 공통)

- **증거 강제:** 모든 판단·등급은 공시의 구체적 근거 인용 필수. 근거 없으면 confidence=low
  + "general knowledge only" 표기.
- **방어가능성 분리:** "공시에서 방어 가능" vs "일반 상식" 구분 표기.
- **낙관 편향 방어:** bear/무효화 조건을 항상 동등 제시. 파괴 리스크·하방 사유 필수화.
- **사람 검수 게이트:** 기회가 가치/선정에 반영되기 전 사람이 등급·근거 확인.

## Phasing & Scope

- **Phase A — DCF 앵커 + 손잡이 노출:** 고정 상수(CAP 5년·터미널 2.5%·WACC)를 명시적·
  재정의 가능한 손잡이로 드러내고 재계산 경로 구축. 방법론 A의 토대.
- **Phase B — 공통 수집(EDGAR 우선) + 싼 이벤트 전처리:** 후보 소싱 깔때기.
- **Phase C — 방법론 A(해자반영 내재가치):** 구조적 thesis 평가 + 등급→손잡이 + 밴드.
- **Phase D — 방법론 B(이벤트 드리븐 기회 thesis):** 정규화 출력.
- **Phase E — 사용자 선택 UI/리포트 + 위험 게이트 연동.**

### 보류 (설계만, 별도 spec)

- **자동 선정 영향(구제/강등):** 기회 thesis가 사람 개입 없이 순위를 바꾸는 경로. 본 spec은
  "사람에게 추천"까지만. 자동 반영은 검증 누적 후 별도 결정.
- **멀티플 점수화 개선:** `valuation_score`의 유니버스 백분위 → 섹터상대 + 상관조정. 별도
  작은 spec.

## 정직한 검증 한계

이건 **투자에서 가장 어려운 일**이고, 액티브 운용 대부분이 비용 빼면 시장에 진다. 우리 자체
발견도 "long-only·공개데이터 시스템은 종이 프리미엄의 ⅓~½만 수확"이다. 따라서:

- **알파 보장 없음.** 만드는 건 "규율 있는 기회 발굴 + thesis 구조화 도구"이지 수익 보장
  기계가 아니다.
- **싼 전처리의 질이 전부를 결정.** 깔때기가 진주를 거르면 LLM이 아무리 좋아도 못 찾는다.
- **정량 백테스트 불가** — point-in-time 공시/뉴스 아카이브 부재 + 장기 판단은 검증에 수년.
  초기 검증 = 소수 종목 수동 검수, 정량 검증 = forward-test 누적.
- "맞는 방법 찾기"는 정량 A/B가 아니라 질적 판단임을 명시. 그래서 다중 방법론 + 사람 게이트.

## Out of Scope

- KR 종목 공시(DART) — 현재 스크리닝 유니버스가 US.
- 자동 매수/매도 신호 — 모든 결정은 사람(철학 §7).
- LLM이 손잡이 숫자·순위 점수를 직접 산출 — 경계선 위반, 금지.
- 자유 점수(0~100) 정성 평가 — 이산 등급만(계산과 판단의 경계 유지).
