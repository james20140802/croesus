# 내재가치: 현재 앵커 + 구조적 thesis 기반 미래 — Design Spec
_2026-06-15_

## Summary

리서치의 산출물을 **"현재 가치 vs 미래 가치의 갭"** 으로 재정의하되, 미래 가치의
무게중심을 **사업 자체에 대한 forward 정성 판단**에 둔다.

기존 시스템은 내재가치 추정의 **현재항만** 구현돼 있고, 그것마저 미래를 과거 FCF의
기계적 외삽으로 때운다. 투자 철학 §3이 "LLM은 뉴스·공시·**경쟁 포지셔닝·산업 내러티브**를
해석하라"고 명시했지만, 구현(Sprint 010)은 LLM을 정량 점수의 재서술로 가둬 놨다.

단순히 기업 발표자료를 정량 가정으로 환원하면 그건 "현재 가치를 정교화"한 것일 뿐이다.
멀티년 수익률을 실제로 가르는 것은 **경쟁우위(해자)·기술력·분야 유망성·파괴 리스크**에
대한 판단이며, 이는 스프레드시트가 못 한다. 이 설계는 그 **구조적 thesis를 1급 산출물로
승격**하고, 이를 장기 DCF 손잡이에 bounded·사람검수 경로로 반영해 "**해자반영 내재가치**"를
산출한다. LLM은 증거에 묶인 "판단"만 하고, 숫자는 전부 결정론적 코드가 계산한다.

---

## Background

### 핵심 통찰 — 두 문제는 하나의 축, 무게는 미래 정성 판단에

원래 별개로 보였던 두 의문 — (1) "왜 LLM이 뉴스를 안 보나?" (2) "PER/PBR 단순 평균이
옳은가?" — 은 **내재가치 추정**의 현재항/미래항이다.

> **내재가치 = 현재항(후행 데이터) + 미래항(forward 정성 판단).
> 미래항의 무게중심은 사업 경쟁력·분야 유망성 판단이다.**

지금은 현재항만 있고, 미래항(그것도 가장 중요한 구조적 판단)이 통째로 비어 있다.

### 실무 정합성 — 그리고 정성 판단이 본체인 이유

펀더멘털 리서치의 실제 루프: 1차 자료 → forward 가정 갱신 → 모델 재계산 → 현재가 비교.
그러나 뛰어난 애널리스트와 스프레드시트를 가르는 건 **근단기 가이던스 숫자가 아니라
장기 경쟁력 판단**이다 — 해자가 넓은가, 기술이 정말 앞서는가, 이 산업이 구조적
성장기인가, 파괴당할 리스크는 없는가. 이 판단이 DCF의 장기 손잡이를 결정한다.

철학 §3이 LLM 용도로 명시한 목록에 *"Competitive positioning", "Product and industry
narratives", "Regulatory risk", "Management commentary"* 가 그대로 있다. 이 설계는 §3을
온전히 구현한다.

### 충돌하는 긴장 (반드시 명시)

같은 철학 §133은 *"Pure LLM-driven stock picking", "Meme-stock recommendations without
fundamental support", "Treating a high factor score as an automatic buy signal"* 를
금지한다. LLM은 산업 전망에서 자신만만하게 틀리고 낙관 편향이 심하다.

**해소:** 구조적 thesis를 허용하되 **증거를 강제**한다(아래 §가드레일). "증거 기반 정성
판단"(§3 허용)이지 "vibes 종목 추천"(§133 금지)이 아니게 만든다.

### 현재 코드의 빈 구멍

`croesus/factors/equity/valuation.py`의 `two_stage_dcf`는 장기 손잡이가 전부 **고정 상수**다:

```python
growth = compute_fcf_growth(calc.annual_fcf)   # 미래 = 과거 FCF 직선 외삽
DCF_EXPLICIT_YEARS = 5                          # 경쟁우위 지속기간 = 고정 5년
DEFAULT_TERMINAL_GROWTH = 0.025                 # 터미널 성장 = 고정 2.5%
# WACC = CAPM 기계값, 파괴 리스크 반영 경로 없음
```

해자가 넓든 좁든, 산업이 성장하든 사양이든 똑같은 손잡이를 쓴다. 구조적 판단이 들어갈
자리가 통째로 비어 있다.

## Architecture

```
[1] 수집 (EDGAR 1차 + 뉴스, 숏리스트 종목만)        → disclosures / news_items
        ↓
[2] LLM 정성 판단 (읽기/판단만, 증거강제)
      ├─ 근단기: 가이던스 등 정량 주장 추출            (보조)
      └─ 구조적 thesis ★ 본체: 해자·기술력·분야유망성·파괴리스크
                       (평점 + 근거 + confidence + 출처)
        ↓  ── 여기까지만 LLM ──
[3] 결정론적 가정 매핑 (bounded · 사람검수)
      ├─ 근단기 주장   → 단기 성장률/마진 델타
      └─ 구조적 thesis → 장기 손잡이: CAP 길이 · 터미널 성장 · 할인율 리스크프리미엄
        ↓
[4] 결정론적 DCF 재계산 → 시나리오 밴드
      현재가치 DCF(기계적)  |  해자반영 DCF(bear/base/bull)
        ↓
[5] 병치 표현 + thesis 카드 → 사람 판단
```

### 경계선 (철학 §2 — LLM은 계산하지 않는다)

- LLM은 [2]에서 **"판단"만** 한다: 정량 주장은 추출, 구조적 차원은 **이산 등급으로 평가**.
  숫자(등급→손잡이 값, 손잡이→가치)는 전부 [3]~[4]의 결정론적 코드가 계산한다.
- 갱신된 가정·손잡이 세트는 [5] 이전에 **사람이 검수**한다.
- **LLM 출력이 순위 점수로 직접 흘러드는 경로는 없다**(선정 영향은 보류).

### [1] 수집 범위

| 자료 | 내용 | 소스 |
|---|---|---|
| 10-K / 10-Q | 재무제표 + MD&A + 리스크 요인 | **SEC EDGAR 무료·구조화(XBRL)** ★ 1차 |
| 8-K | 수시공시, 실적 보도자료(첨부), 가이던스 | EDGAR ★ |
| IR 프레젠테이션 / 어닝콜 transcript | 전략·제품 로드맵·경쟁구도·Q&A | 일부 유료/스크래핑 (best-effort) |
| 뉴스 헤드라인 | 2차·보조 | 뉴스 API (보조) |

EDGAR가 1차 — 권위 있는 기업 자신의 공시, 무료, 기계 판독 가능. 특히 10-K의 "Business",
"Risk Factors", MD&A 섹션이 경쟁력·산업·리스크 판단의 1차 근거다.

### [2] 구조적 thesis — 본체 (이산 등급 + 증거)

LLM은 각 차원을 **고정된 이산 등급**으로 평가하고, 각 등급마다 **공시의 구체적 근거**를
인용한다(자유 점수 아님 — 계산이 아니라 분류):

| 차원 | 등급 | 근거 예시(공시에서) |
|---|---|---|
| 경쟁우위 / 해자 | wide / narrow / none | 전환비용, 네트워크 효과, 규모, IP·특허, 브랜드 |
| 기술력 | leading / parity / lagging | R&D 집약도, 제품 로드맵, 벤치마크·점유율 |
| 분야 유망성 | secular_growth / stable / declining | TAM 성장, 산업 데이터 |
| 파괴 리스크 | low / medium / high | 신기술, 신규진입자, 규제 |

각 평가에 **confidence(high/med/low)** 와 **provenance**("공시에서 방어 가능" vs "일반
상식")를 붙인다. → `structural_thesis` 테이블에 저장.

### [3] 매핑 — 등급을 DCF 손잡이로 (우리가 소유한 고정·bounded 테이블)

**근단기(정량 주장):** 가이던스 숫자를 출처와 함께 추출 → 단기 성장률에 대입·클램프.

```python
base_growth = clamp(guided_growth, FCF_GROWTH_FLOOR, FCF_GROWTH_CAP)  # 기존 -0.05~0.30
```

**구조적 thesis → 장기 손잡이** (예시 테이블, 전부 우리가 정한 상수, bounded):

```python
# 경쟁우위 지속기간 (CAP) — 현재 고정 5년을 등급별로
CAP_YEARS = {"wide": 10, "narrow": 7, "none": 5}        # 해자 등급
# 터미널 성장 — 해자·산업 유망성이 GDP초과 영구성장을 정당화하는가
TERMINAL_GROWTH = {"secular_growth": 0.030, "stable": 0.025, "declining": 0.015}
# 할인율 리스크프리미엄 — 파괴 리스크
RISK_PREMIUM = {"low": 0.00, "medium": 0.01, "high": 0.02}  # WACC에 가산
```

LLM은 등급만 고르고, 등급→숫자는 감사가능한 우리 테이블이다. 전부 기존 캡으로 바운드되고
사람이 검수한다. **정밀 추정이 아니라 시나리오 노브임을 명시.**

### [4] 시나리오 밴드

- **현재가치 DCF**: 손잡이 전부 기계적 기본값(기존 동작). 비교의 닻.
- **해자반영 DCF (base)**: 근단기 가이던스 + 구조적 thesis의 base 등급 반영.
- **bull / bear**: thesis 등급을 한 단계 위/아래로(예: 해자 wide→narrow) 흔든 밴드 →
  "이 판단이 틀리면 가치가 이만큼 움직인다"의 폭.

### [5] 병치 표현 + thesis 카드

리포트/리서치노트에:
- 비교 블록: `현재가 | 현재가치 DCF | 해자반영 DCF (bear/base/bull)`
- **thesis 카드**: 4개 차원 등급 + 근거 인용 + confidence + 출처 링크.

비교·판단은 사람이 한다 — 자동 매수/매도 신호 없음(철학 §7).

### 가드레일 (§133 vibes-picking 방지)

- **증거 강제**: 모든 등급은 공시의 구체적 근거 인용 필수. 근거 없으면 등급 부여 금지(→
  confidence=low + "general knowledge only" 표기).
- **방어가능성 분리**: "공시에서 방어 가능" vs "일반 상식"을 차원별로 구분 표기.
- **낙관 편향 방어**: bear 시나리오를 항상 동등 비중으로 제시. 파괴 리스크 차원을 필수화.
- **사람 검수 게이트**: 손잡이가 가치에 반영되기 전 사람이 등급·근거를 확인.

## Phasing & Scope

- **Phase A — DCF 앵커 + 손잡이 노출**: 고정 상수(CAP 5년·터미널 2.5%·WACC)를 **명시적·
  재정의 가능한 손잡이**로 드러내고 재계산 경로를 만든다. 구조적 thesis 없이도 동작.
- **Phase B — 수집(EDGAR 우선) + LLM 정성 판단**: 근단기 추출 + 구조적 thesis 평가.
- **Phase C — 매핑 + 해자반영 DCF + 시나리오 밴드**: 고정 테이블, bounded, 사람검수.
- **Phase D — 병치 비교 블록 + thesis 카드**: 리포트/리서치노트.

### 보류 (설계만, 별도 spec)

- **선정 영향(구제/강등)**: 해자반영 가치/thesis 등급이 종목 순위를 바꾸는 경로.
  Phase A~D를 사람검수 우선으로 구현·검증한 뒤 결정. 전 유니버스 평가가 필요해 비용 큼.
- **멀티플 점수화 개선** (별도 작은 spec): `valuation_score`의 유니버스 백분위 →
  섹터상대(`pe_vs_sector_pct` 이미 계산됨) + 상관조정(PER·PBR 이중계산 완화).

## 정직한 검증 한계

구조적 thesis·해자반영 가치는 **014 같은 정량 백테스트가 불가**하다 — point-in-time
공시/뉴스 아카이브가 있어야 "과거에 이렇게 판단했으면 맞았나"를 검증하는데 비싸거나
구하기 어렵고, 장기 경쟁력 판단은 검증에 수년이 걸린다. 따라서:

- **초기 검증 = 소수 종목 수동 검수**: 등급 근거가 실재 공시에 있는가, 낙관 편향은 없는가,
  해자반영 가치가 말이 되는가.
- **정량 검증 = forward-test 누적**: 시간을 들여 out-of-sample로 쌓는다.
- "맞는 방법 찾기"는 정량 A/B가 아니라 질적 판단임을 명시한다. 이게 이 영역의 본질적
  한계이며, 그래서 사람 검수 게이트가 필수다.

## Out of Scope

- KR 종목 공시(DART) — 현재 스크리닝 유니버스가 US.
- 자동 매수/매도 신호 — 모든 비교는 사람 판단(철학 §7).
- LLM이 손잡이 숫자나 순위 점수를 직접 산출 — 경계선 위반, 금지.
- 자유 점수(0~100) 평가 — 이산 등급만(계산과 판단의 경계 유지).
