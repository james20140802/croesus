# 내재가치: 현재 앵커 + 뉴스 기반 미래 — Design Spec
_2026-06-15_

## Summary

리서치의 산출물을 **"현재 가치 vs 미래 가치의 갭"** 으로 재정의한다.

기존 시스템은 내재가치 추정의 **현재항만** 구현돼 있고, 그것마저 미래를 과거 FCF의
기계적 외삽으로 때운다. 투자 철학 문서 §3이 "LLM은 뉴스·공시·실적콜을 해석하라"고
약속했지만 구현(Sprint 010 Research Agent)은 LLM을 "이미 계산된 정량 점수의 재서술"로
가둬 놨다 — 프롬프트가 `do not claim knowledge of recent news`를 명시한다.

이 설계는 **기업 1차 공시(SEC EDGAR) + 뉴스**를 수집해 forward 정보를 구조화 추출하고,
이를 결정론적으로 DCF 가정에 반영해 **현재가 vs 기계적 DCF vs 뉴스반영 DCF 밴드**를
나란히 제시한다. LLM은 "읽기"만 하고, 숫자는 전부 결정론적 코드가 계산한다.

---

## Background

### 핵심 통찰 — 두 문제는 하나의 축이다

원래 별개로 보였던 두 의문 —
1. "왜 LLM이 뉴스를 안 보고 정량 점수만 재서술하는가?"
2. "PER/PBR를 단순 백분위 평균하는 게 옳은가?"

— 은 **내재가치 추정**이라는 한 축의 현재항/미래항이다.

> **내재가치 = 현재항(후행 데이터 기반) + 미래항(forward 정보 기반).
> 둘의 갭이 리서치의 산출물.**

지금은 현재항만 있고(미래를 과거의 외삽으로 대체), 미래항이 통째로 비어 있다.

### 실무 정합성

펀더멘털 리서치(sell-side/buy-side)의 실제 루프와 일치한다:
1. 기업 1차 자료를 읽고 → 2. 모델의 forward 가정(매출성장·마진 등)을 갱신하고 →
3. 모델(3-statement + DCF)이 가치를 재계산 → 4. 내재가치를 현재가와 비교.

**중요한 차이(정직):** 기관은 3번의 "주장→가정" 변환을 사람의 판단으로 한다.
이 설계는 그 자리에 LLM 추출 + 결정론적 매핑을 넣는다 — 거기가 가장 어려운 지점이며,
정성 주장의 매핑은 "정밀 추정"이 아니라 "시나리오 노브"로만 다룬다(아래 §경계선).

### 현재 코드의 빈 구멍

`croesus/factors/equity/valuation.py`의 `two_stage_dcf`는 미래 성장을 이렇게 만든다:

```python
growth = compute_fcf_growth(calc.annual_fcf)   # 과거 3~5년 FCF의 기계적 외삽
```

즉 DCF의 "미래"가 과거의 직선 연장일 뿐이다. forward 정보가 들어갈 자리가 비어 있다.
`valuation_snapshots.assumptions` dict에 가정이 절반쯤 기록돼 있으나, 재정의·재계산
경로가 없다.

## Architecture

```
[1] 수집 (신규 ingestion job)        SEC EDGAR(1차 공시) + 뉴스 API → news_items / disclosures
        ↓                            ※ 철학 §4: 숏리스트 종목에만 (비용/스코프 제한)
[2] forward 주장 추출 (로컬 LLM)      자료 → 구조화 주장 + 출처링크 → forward_claims
        ↓  ── 여기까지만 LLM ──
[3] 가정 갱신 매핑 (결정론적 코드)     주장 → DCF 가정 델타 (기존 캡으로 바운드)
        ↓
[4] 시나리오 재계산 (결정론적 DCF)     bear/base/bull 가정 → 내재가치 밴드 → valuation_scenarios
        ↓
[5] 병치 표현                        현재가 vs 기계적DCF vs 뉴스반영DCF밴드 → 사람 판단
```

### 경계선 (철학 §2 유지 — LLM은 계산하지 않는다)

- LLM은 **[2]에서 "읽기"만** 한다: 자료에서 forward 주장을 구조화 추출(출처 링크 필수).
- 숫자는 전부 [3]~[4]의 결정론적 코드가 계산한다.
- 갱신된 가정 세트는 [5] 이전에 **사람이 검수**한다.
- **LLM 출력이 순위 점수로 직접 흘러드는 경로는 없다.**

### [1] 수집 범위

| 자료 | 내용 | 소스 |
|---|---|---|
| 10-K / 10-Q | 재무제표 + MD&A | **SEC EDGAR 무료·구조화(XBRL)** ★ 1차 |
| 8-K | 수시공시, 실적 보도자료(첨부) | EDGAR ★ |
| 가이던스 | "내년 매출 12~15% 성장 전망" 등 정량 | 8-K 첨부 |
| IR 프레젠테이션 / 어닝콜 transcript | 전략·제품 로드맵·Q&A | 일부 유료/스크래핑 (후순위) |
| 뉴스 헤드라인 | 2차·보조 | 뉴스 API (보조) |

EDGAR를 1차 소스로 삼는다 — 권위 있는 기업 자신의 공시, 무료, 기계 판독 가능.
(현재 스크리닝 유니버스가 US라 EDGAR; KR 확장 시 DART가 동형 역할.)

### [2]→[3] 주장→가정 델타 매핑 (가장 약한 고리, 두 갈래)

**경우 A — 기업이 이미 숫자를 준 경우 (튼튼함, 가치의 대부분)**

가이던스는 대부분 이미 정량적이다. LLM은 숫자를 출처와 함께 **추출만** 한다.
[3]은 대입 + 바운드:

```python
# guided_growth = 0.135 (LLM이 8-K에서 추출; 출처: "FY24 guidance, 2024-01-25")
base_growth = clamp(guided_growth, FCF_GROWTH_FLOOR, FCF_GROWTH_CAP)  # 기존 -0.05~0.30 재사용
```

매출성장 → FCF성장 변환이 필요하면 그 변환식(예: 마진 일정 가정)도 결정론적 코드에 둔다.

**경우 B — 정성적 주장 (숫자 없음 → 자의적, 시나리오로만)**

LLM은 **방향 × 강도 버킷으로 분류만** 한다(계산 아님):

```
claim: "신규 데이터센터 규제로 매출 risk"
  → direction = negative, magnitude = medium   # LLM 선택은 여기까지
```

버킷→숫자 변환표는 우리가 고정·소유한다:

```python
MAGNITUDE_DELTA = {"small": 0.02, "medium": 0.05, "large": 0.10}  # 우리가 정한 상수
delta = sign(direction) * MAGNITUDE_DELTA[magnitude]
```

**정직한 한계:** 이 매핑은 본질적으로 자의적이다. "정밀 추정"이 아니라 시나리오 노브다.

### [4] 시나리오 밴드

- **base** = 경우 A(가이던스 정량)만 반영, 정성 주장 무시 → 가장 방어 가능한 추정
- **bull / bear** = 경우 B 정성 델타까지 적용 → "뉴스가 맞다면 가치가 이만큼 움직인다"의 폭

각 시나리오는 결정론적 DCF를 다시 돌려 내재가치를 산출한다.

### [5] 병치 표현

리포트/리서치노트에 비교 블록 추가: `현재가 | 기계적 DCF | 뉴스반영 DCF (bear/base/bull)`.
비교(갭 판단)는 사람이 한다 — 자동 매수/매도 신호 없음.

## Phasing & Scope

- **Phase A — DCF 앵커 강화 (현재가치 먼저, 합의된 우선순위)**:
  `compute_fcf_growth`의 기계적 외삽을 **명시적·재정의 가능한 가정**으로 드러내고
  재계산 경로를 만든다. `valuation_snapshots.assumptions`에 절반 존재 → 확장.
- **Phase B — 수집 + forward 주장 추출**: EDGAR 우선, 뉴스 보조.
- **Phase C — 주장→가정 매핑 + 시나리오 밴드**: 경우 A/B, 고정 변환표.
- **Phase D — 병치 비교 블록**: 리포트/리서치노트.

### 보류 (설계만, 별도 spec)

- **선정 영향(구제/강등)**: 미래가치가 종목 순위를 바꾸는 경로. Phase A~D를 주석 우선으로
  구현·검증한 뒤 결정. 전 유니버스 DCF가 필요해 비용이 큼.
- **멀티플 점수화 개선** (별도 작은 spec): `valuation_score`의 유니버스 백분위 →
  섹터상대(`pe_vs_sector_pct` 이미 계산됨) + 상관조정(PER·PBR 동일가중 이중계산 완화).

## 정직한 검증 한계

뉴스반영 가치는 **014 같은 정량 백테스트가 불가**하다 — point-in-time 뉴스/공시
아카이브가 있어야 "과거에 이 자료로 가정을 바꿨으면 맞았나"를 검증하는데, 비싸거나
구하기 어렵다. 따라서:

- **초기 검증 = 소수 종목 수동 검수**: LLM 추출이 헛소리 안 하는가, 출처가 실재하는가,
  갱신된 가정·갭이 말이 되는가.
- **정량 검증 = forward-test 누적**: 시간을 들여 out-of-sample로 쌓는다.
- "맞는 방법 찾기"는 정량 A/B가 아니라 질적 판단임을 명시한다.

## Out of Scope

- KR 종목 공시(DART) — 현재 스크리닝 유니버스가 US.
- 어닝콜 transcript 실시간 수집(유료) — 후순위, Phase B에서 best-effort.
- 자동 매수/매도 신호 — 모든 비교는 사람 판단(철학 §7).
- LLM이 가정 숫자를 직접 산출 — 경계선 위반, 금지.
