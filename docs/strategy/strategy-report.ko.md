# Croesus 투자 전략 보고서

이 문서는 (1) **사용자 입장에서 프로그램이 어떤 흐름으로 도는지**, (2) **지금까지
테스트한 모든 전략의 설명·결과·인사이트**, (3) 그 바탕이 되는 **팩터 모델**을 한국어로
정리한 보고서입니다.

기술 레퍼런스: `docs/strategy/multifactor-model.md` (팩터별 input/process/output 영문),
`docs/operations/backtest.md`, `docs/operations/forward-test.md`.

---

## 1. 사용자 관점 — 프로그램은 어떻게 도는가

Croesus는 **"넓게 정량 스크리닝 → 좁혀서 정성 리서치 → 인간 승인 → (선택) 실행"** 순서로
도는 CLI 파이프라인입니다. 트레이드는 **사용자가 명시적으로 승인하기 전엔 절대 실행되지
않습니다.**

### 1단계 — 온보딩 (최초 1회)
```
python -m croesus.jobs.profile_init       # 위험성향·목표수익·감내낙폭 등 프로필 설정
python -m croesus.jobs.import_holdings     # 보유 CSV 임포트 (심볼 자동 등록)
```
- 프로필이 위험-수익 정합성을 검증합니다(예: 목표 10%인데 감내낙폭 -5%면 경고).
- 감내낙폭은 **음수로 저장**(예: -0.25 = "25% 하락까지 감내").

### 2단계 — 매일 도는 파이프라인 (`local_sync` 한 줄)
```
python -m croesus.jobs.local_sync          # 아래 잡들을 신선도 기반으로 순서대로 실행
python -m croesus.jobs.local_sync --status # 한눈에 상태 (READY/DEGRADED/STALE)
```
내부적으로 이 순서로 흐릅니다:

| 순서 | 잡 | 하는 일 (input → output) |
|---|---|---|
| 1 | `daily/weekly/monthly_macro_run` | FRED 지표 → 3층 매크로 점수(레짐/증폭/확인) |
| 2 | `universe_refresh` (주간) | Wikipedia S&P500+NASDAQ100 → 유니버스(~600종목) |
| 3 | `daily_run` | yfinance 가격 → **공통 팩터**(모멘텀/변동성/유동성/200일선/**베타**) |
| 4 | `quarterly_run` | 재무제표 → **밸류에이션**(P/E·P/B·DCF) + **퀄리티**(ROE·마진·레버리지) |
| 5 | `portfolio_snapshot` | 보유 × 시가 × 환율 → 평가액·슬리브·집중도 노출 (결손은 DEGRADED) |
| 6 | `screening_run` | 팩터 × 레짐 가중 → **종목 랭킹 리포트** (차원 분해 + 밸류에이션) |
| 7 | `rebalance_check` | 노출 위반 → **리밸런싱 제안**(trim/raise_cash/block-buy) + 신규후보 리서치 노트 |
| 8 | `performance_check` | 스냅샷 이력 → 목표 대비 성과 + 리스크 판정(근거 명시) |

핵심 원칙:
- **데이터 결손은 조용히 넘어가지 않습니다.** 가격/환율 누락은 data-quality 오류로 기록되고
  상태가 DEGRADED로 떨어지며 리포트 최상단에 표기됩니다.
- 매크로 레짐은 스크리닝 **가중치만** 조정합니다(일방향 의존).
- 보유 중 같은 회사·지수 중복(GOOG/GOOGL, SPY/VOO)은 합산 비중이 단일 포지션 한도를
  넘으면 위반으로 잡혀 trim 제안이 나옵니다.

### 3단계 — 승인 → (선택) Paper 실행
```
python -m croesus.jobs.list_pending_approvals          # 대기 제안 목록
python -m croesus.jobs.approve_action <id>             # 건별 승인(7일 만료)
python -m croesus.jobs.execute_approved <id> --dry-run # 주문 미리보기
python -m croesus.jobs.execute_approved <id>           # paper 체결 → 원장 기록
```
- 승인 없이는 어떤 실행 경로도 존재하지 않습니다. 만료·미승인·이미체결은 모두 차단됩니다.
- 슬리브 레벨 제안(raise_cash 등)은 "주문이 아니라 목표"라서 실행에서 스킵됩니다.

### 4단계 — 실험 트랙 (수동, 파이프라인 밖)
```
python -m croesus.jobs.backtest_run --start 2021-01-01 --end 2026-06-01 --top-n 10 --buffer 2.0
python -m croesus.jobs.forward_test_run --record       # 오늘의 전략별 코호트 기록
python -m croesus.jobs.forward_test_run --evaluate --report
```
- **백테스트**: 과거 가격으로 규칙을 워크포워드 재생(가격 팩터만 — 밸류/퀄리티는 look-ahead라 제외).
- **forward-test**: 오늘 살 종목을 기록 → 앞으로 실현수익을 SPY와 비교(밸류/퀄리티 검증의 유일한 정직한 방법).

## 2. 점수는 어떻게 매겨지나 (팩터 모델 요약)

각 종목은 7개 차원의 **백분위 점수(0~1)**를 가중 합산해 점수를 받습니다. 데이터가 없는
차원은 **가중치가 재정규화되어 사라집니다**(없다고 0점이 아님).

| 차원 | 원천 | 방향 | 백테스트? |
|---|---|---|---|
| 모멘텀 (1/3/6개월) | 가격 | 높을수록 ↑ | ✅ |
| 추세 (200일선 위) | 가격 | 게이트/가점 | ✅ |
| 유동성 (거래대금) | 가격 | 높을수록 ↑ | ✅ |
| 변동성 페널티 | 가격 | 낮을수록 ↑(차감) | ✅ |
| **저베타** (SPY 대비 베타) | 가격 | 낮을수록 ↑(반전) | ✅ |
| 밸류에이션 (P/E·P/B·DCF) | 펀더멘털 | 쌀수록 ↑(반전) | ❌ (look-ahead) |
| **퀄리티** (ROE·마진·레버리지) | 펀더멘털 | 우량할수록 ↑ | ❌ (look-ahead) |

선정 후 **churn 히스테리시스**(순위 밴드 안이면 유지)와 **중복 그룹 캡**(GOOG+GOOGL =
1슬롯)을 적용해 포트폴리오를 구성합니다.

## 3. 우리가 테스트한 모든 전략

### 3-A. 백테스트로 검증 가능한 전략 (가격 팩터만)

**공통 조건**: 2021-01 → 2026-06 (5.4년), top-10, 히스테리시스 buffer 2.0, 비용 10bps,
월별 리밸런스, 벤치마크 SPY. (생존편향·밸류/퀄리티 제외 한계는 §4 참조.)

| 전략 | 가중치 | CAGR | Sharpe | MDD | 누적수익 | 회전율 |
|---|---|---|---|---|---|---|
| **composite_v1** | 모멘텀.35/유동성.25/추세.25/변동성-.15 | **13.13%** | **0.81** | **-16.4%** | 94.8% | 33 |
| momentum_tilt | 모멘텀.60/추세.15/변동성-.25 | 11.42% | 0.78 | -17.2% | 79.4% | 44 |
| multifactor_lowbeta | 모멘텀.45/추세.15/**저베타.25**/변동성-.15 | 6.69% | 0.50 | -22.4% | 41.9% | 36 |
| momentum_only | 모멘텀 1.0 | 40.98% | 1.20 | **-34.3%** | 539.8% | 45 |
| **SPY (매수보유)** | — | 14.28% | 0.88 | -25.4% | 105.7% | 0 |

**수치 의미**: CAGR=연복리수익 · Sharpe=위험 대비 수익(높을수록 효율적) ·
MDD=최대낙폭(덜 음수일수록 얕음) · 회전율=누적 거래 비율(비용 대용).

#### composite_v1 — 추세 타는 우량주
- **설명**: 모멘텀+유동성+추세+저변동의 균형 블렌드. 사실상 "추세를 타는 우량 대형주".
- **결과**: risk-adjusted 최고(Sharpe 0.81). SPY보다 **낙폭이 훨씬 얕음**(-16% vs -25%).
- **insight**: 처음엔 -6.6%/-1.06로 "처참"했는데, **신호가 아니라 구성이 망가져 있었습니다.**
  종목 수 5→10, churn 억제, 중복 캡 세 가지를 고치자 13.1%로 반전. SPY 절대수익(14.3%)은
  근소하게 못 이김 → **"저위험 SPY"**이지 아직 알파원은 아님.

#### momentum_tilt — 낙폭 인식 모멘텀
- **설명**: 모멘텀을 키우되 변동성 페널티를 0.25로 강화. 유동성은 게이트로만.
- **결과**: composite_v1과 비슷하나 약간 열위(11.4%/0.78).
- **insight**: 작은 N(top-5)에선 composite_v1보다 견고했지만, top-10+buffer에선 역전당함.
  변동성 페널티가 낙폭은 잘 잡지만 회전율(44)이 높음.

#### multifactor_lowbeta — 저베타(BAB) 추가
- **설명**: composite에 저베타 차원(0.25)을 넣어 방어력 강화를 노림.
- **결과**: **모든 축에서 최악**(6.7%/0.50/-22.4%). 방어 팩터인데 낙폭이 *더 깊어짐*.
- **insight (중요)**: 저베타가 이 구간에서 **역풍**이었습니다. ① 2021–26은 메가캡/AI 모멘텀
  장세 → 저베타는 정의상 그 승자를 비켜감(문서화된 BAB의 부진한 10년). ② "저베타"가
  시장과 *무상관*이지만 *개별 변동성은 큰* 에너지·가치주를 샀음 → **시장 베타 낮음 ≠ 낙폭
  낮음**. ③ 실제로 통했던 추세 보호를 희석. **단일 팩터의 가치는 레짐 의존적**이라는
  교과서적 사례. 저베타는 시장 *조정기*에 값을 하도록 설계됐는데 이 구간엔 그게 없었음.

#### momentum_only — 순수 모멘텀
- **설명**: 모멘텀 100%. 집중 베팅.
- **결과**: 압도적 raw 수익(+540%, CAGR 41%)이지만 낙폭 -34.3%로 **프로필 한도(-25%) 위반**.
- **insight**: +540%의 상당 부분은 **생존편향 신기루**(오늘의 S&P500 승자 풀에서 모멘텀
  top을 뽑음). 방향(모멘텀 강세)은 신뢰해도 절대 수치는 신뢰 금지. 리스크 감내 의향이 있는
  사용자를 위해 forward-test의 `momentum_aggressive`로 **실현 낙폭을 라이브 관찰** 후 결정.

### 3-B. forward-test로만 검증 가능한 전략 (밸류·퀄리티 포함)

밸류에이션·퀄리티는 과거 시점 펀더멘털이 없어 백테스트하면 look-ahead입니다. 그래서 **오늘
살 종목을 기록 → 앞으로 SPY와 비교**합니다. 첫 코호트 2026-06-12 기록, **실현수익은 몇 달
쌓여야 의미**가 있습니다. 아래는 각 전략이 *지금* 고른 종목으로, **성격이 가중치대로 또렷이
갈린다는 증거**입니다.

| 전략 | 가중치 | 오늘의 대표 종목 | 성격 |
|---|---|---|---|
| composite_live | 라이브 기본(밸류.10) | C, SPY, KLAC, BAC, VOO, JNJ | 혼합(기준선) |
| **composite_v2_value** | 밸류.30 틸트 | STT, MET, BNY, PFG, CB, VZ | **싼 금융·보험·통신** |
| **composite_v3_multifactor** | 모멘텀.25/밸류.25/퀄리티.20/저베타.15/추세.15 | MO, APA, ALL, EOG, CB, CF, TRV | **방어적 가치-퀄리티**(싼 보험사 + 음의 베타 에너지) |
| momentum_aggressive | 모멘텀.85/추세.15 | MRVL, MU, DELL, ARM, HPE | **AI 하드웨어 모멘텀** |

- **composite_v2_value**: 밸류 가중을 0.10→0.30으로 올리자 싼 보험·은행으로 완전히 이동.
  가치는 모멘텀과 음상관(≈-0.6)이라 블렌드 시 위험 분산 기대.
- **composite_v3_multifactor**: 가치+퀄리티+저베타 → 싼 보험사(ALL P/E 5.8)와 음의 베타
  에너지(APA·EOG)로 수렴. **시장과 무관하게 싸고·수익성 좋은** 종목들. 연구가 가장 방어
  가능하다고 본 조합.
- **momentum_aggressive**: 반도체·AI 하드웨어로 도배. raw 수익 잠재 최고, 낙폭도 최고 예상.

## 4. 종합 인사이트

1. **구성(construction)이 신호(signal)보다 중요했다.** composite_v1을 -6.6% 재앙에서
   +13.1%로 바꾼 건 가중치가 아니라 종목 수·churn·중복 처리였습니다. 분산이 유일한 공짜
   점심이고, 잡음 매매(churn)는 비용일 뿐 아니라 모멘텀을 잘라 알파를 파괴합니다.

2. **단일 팩터의 가치는 레짐 의존적이다.** 저베타가 2021–26 모멘텀 장세에선 역풍이었듯,
   어떤 팩터도 한 구간으로 판정하면 안 됩니다. **그래서 블렌드**하고, **그래서 forward-test로
   여러 레짐을 관찰**합니다.

3. **아직 SPY를 못 이긴다 — 정직하게.** 제대로 구성한 composite_v1도 risk-adjusted로는
   비등하지만 절대수익은 근소하게 밑입니다. 메가캡 지수가 지배한 5년에선 예상 가능한 결과.
   초과수익은 이 백테스트에서 빠진 **밸류에이션·퀄리티·동적 매크로 틸트**에서 나와야 하고,
   그건 forward-test로만 정직하게 검증됩니다.

4. **연구가 말하는 현실**: 무레버리지·롱온리·공개데이터·대형주 시스템이 실제로 수확 가능한
   건 (a) 비용·회전율 최소화(확정), (b) 가치+모멘텀+퀄리티+저베타 **결합**(논문 프리미엄의
   1/3~1/2만 포착), (c) 세금 인지 정도입니다. 속도·대체데이터·사모는 구조적으로 불가.
   초과수익의 대부분은 "위험 보상"이지 마법이 아닙니다.

5. **남은 한계**: 생존편향(오늘의 지수 구성종목)이 모든 백테스트를 부풀립니다. 밸류·퀄리티는
   백테스트 불가(forward-test 누적 필요). 베타는 불안정·레짐 의존적입니다.
